# Issues

Issues are problems, opportunities, and ideas tracked through IDS (Identify, Discuss, Solve).

## Issue list

**Path:** `/org/[slug]/teams/[teamSlug]/issues`

### Create an issue

1. Open team **Issues** (or **Create → Issue**).
2. Enter title and description.
3. Set **owner**, **priority** (1 = highest), and optional links.

### Priority

Issues are sorted by priority number. Lower numbers surface first in L10 IDS.

Drag and drop to reorder priorities when permitted.

### Parking lot

Issues marked **parking lot** are deferred — not for today's IDS but kept visible for future weeks.

- Toggle parking lot from the issue detail or list.
- Parking lot issues appear separately during L10 IDS.

### Archive

Completed or irrelevant issues can be archived to keep the active list focused.

---

## IDS in L10

During the **Issues** section of a live L10:

1. Review the prioritized list (Top 3 mode available).
2. Discuss one issue at a time.
3. Solve or assign next steps (often becomes a **to-do**).
4. Move non-urgent items to the **parking lot**.

---

## AI duplicate finder

On the issues workspace, run **Find duplicates** to get AI suggestions for merging similar issues.

Review each suggestion in the approval panel before merging. See [AI Assistant](./ai-assistant.md).

---

## Best practices

- Capture issues all week; solve them in L10.
- Keep the active list short — use parking lot liberally.
- Every issue has one owner.
- End IDS with clear to-dos, not vague follow-ups.

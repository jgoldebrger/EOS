# Projects

Projects extend EOS with structured work tracking — work items, cycles, and labels alongside your traction tools.

## Project list

**Path:** `/org/[slug]/projects`

### Create a project

1. Go to **Projects** (or **Create → Project**).
2. Enter name, optional team, and project lead.
3. Open the project workspace.

### Filters

Filter by status, team, or archive state.

---

## Project workspace

**Path:** `/org/[slug]/projects/[projectSlug]`

### Work items

Tasks within the project with status, assignee, and priority.

### Views

- **Table** — spreadsheet-style list.
- **Kanban** — board by status column.

### Cycles

Optional time-boxed cycles (sprints or phases) to group work items.

### Labels

Color-coded labels for cross-cutting tags (bug, feature, urgent).

### EOS links

Link work items to existing **issues**, **rocks**, or **to-dos** so project work stays connected to EOS traction.

### Analytics

Summary charts for throughput and completion on the project detail page.

---

## Best practices

- Use projects for multi-week initiatives; use **to-dos** for 7-day actions.
- Link project items to rocks when they support quarterly priorities.
- Archive completed projects to keep the list manageable.
